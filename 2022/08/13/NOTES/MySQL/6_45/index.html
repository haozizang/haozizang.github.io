锁

  1. 全局锁

    • 全局读锁: FTWRL(flush tables with read lock)
      • 整个库会成为只读状态
      • 用于全库备份
        • set readonly=true 也是一个手段(不建议)
        • mysqldump 工具也可完成全库备份(事务方式), 但InnoDB以外的引擎无事务…

  2. 表级锁

    • 表锁

      • 一般是数据引擎不支持行锁才用表锁
        • MyISAM -> InnoDB
      • 读写锁
        • lock tables t1 read, t2 write;

    • 元数据锁(MDL: meta data lock)

      • MDL 默认添加的
      • 事务中的 MDL在事务完成后才释放
      • 读写锁类型
        • 只有对表结构修改时加写锁
        • CRUD是加读锁

  3. 行锁

如何安全给小表加字段?

  • 问题
    • 有 CRUD sess 1 拿着读锁;
    • alter table sess get blocked
    • CRUD sess 2 get blocked
    • 当很多CRUD sess阻塞时库线程暴满
  • 解决
    • 解决长事务(事务不提交会占着 MDL 锁)
    • 对于读写频繁的热点表, 使用超时失效的修改表结构语句; 拿不到锁页不要一直阻塞
      • alter table T NOWAIT...
