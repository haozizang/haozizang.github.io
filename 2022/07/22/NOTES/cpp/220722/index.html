源码-容器

  1. stack

    • 底层默认 Sequence 是从 deque 修改过来的
    • 不提供迭代器

  2. queue

    • 不提供迭代器
    • 底层默认 Sequence 是从 deque 修改过来的

  3. map/set

    • 递增 -> 红黑树实现
    • set 的迭代器是 constance_iterators, 不能修改键
    • set 保存了两份(key/value 相等? why? )

  4. 树节点

    1. 无序树/有序树
      • 任意节点的子节点 有/无 顺序关系
    2. 二叉树
      • 树的任意节点至多包含两棵子树
      • 遍历方法: 前序/中序/后序/层次
      • 完美二叉树: 叶子节点都在同一层; 且除叶子节点外所有节点都有俩子节点
      • 完全二叉树(<=完美二叉树): 除最深层(d层)外的所有节点构成满二叉树, 且d层所有节点从左向右连续紧密排列
      • 完满二叉树(<=完美二叉树): 且除叶子节点外所有节点都有俩子节点
    3. 霍夫曼树
      • 加权路径最短的二叉树
    4. 二叉查找树(BST)
      • 左nodes < 根 node < 右nodes
      • 递归性: 左右两个子树也都是BST
      • 无相等节点
    5. 平衡二叉树
      • 左右两个子树高度差不超过 1
      • 递归性: 左右两个子树都是平衡二叉树
    6. AVL树
      • BST
      • 平衡: 左右子树高度差不超过 1
      • 使用场景
        • 增删少, 查找多
        • 旋转: 降低树的高度, 再平衡
    7. 红黑树
      • BST
      • 节点非黑即红
      • 根节点为黑
      • 红色节点两个子节点都是黑色
      • 从任一节点到其每个叶子的所有路径都包含相同数目的黑色节点
      • 这些约束确保：从根到叶子的最长的可能路径不多于最短的可能路径的两倍长
      • 使用场景
        • 增删也多, 查找多
        • STL中的 map / set; linux 进程调度(completely fair scheduler)
        • epoll 内核中用 RBT 管理事件块
        • nginx 红黑树管理 timer
      • 和 AVL对比:
        • 不严格平衡, 增删快;
        • 旋转次数3此内解决;
        • 查找 AVL 快
    8. 伸展树
      • BST
      • O(logn) 内完成增删查
    9. B-tree (m阶)
      • 节点最多m个子节点
      • 根节点(若非叶)最少2个子
      • 非叶子节点(非根)最少 [m/2向上取整] 个子节点
      • 非根子节点度数k为: m/2 <= k <= m
      • 非根子节点 键数 j: j = k - 1
      • 叶子节点位于同一层(平衡因子 = 0)
      • 特性
        • 自平衡
        • 多路(降低了深度): 适合磁盘数据读取(IO次数少)
        • O(logN)
    10. B+
      • 数据都放在叶子节点, 非叶子节点只存索引
    11. Trie树(字典树)
      • 根节点不包含字符，除根节点外每一个节点都只包含一个字符
      • 从根节点到某一节点，路径上经过的字符连接起来，为该节点对应的字符串
      • 每个节点的所有子节点包含的字符都不相同
      • 应用
        • 字符串查找
        • 搜索引擎
