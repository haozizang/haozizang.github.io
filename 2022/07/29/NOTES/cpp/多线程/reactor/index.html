reactor

  1. 阻塞/非阻塞

    • 无数据时:
      • 阻塞: read 时无数据则等待
      • 非阻塞(reactor): read 时无数据立即返回; 通过不断轮询来获取数据
    • 有数据时:
      • 二者都是同步

  2. 异步 I/O (proactor采用它)

    • aio_read 立即返回
    • 数据准备好(内核自动将数据从内核空间拷贝到用户空间)后通知

  3. reactor vs proactor

    • Reactor是非阻塞同步网络模式，感知的是就绪可读写事件
    • Proactor是异步网络模式，感知的是已完成的读写事件

  4. reactor 模式的应用

    • redis
    • nginx
    • netty

  5. reactor组成

    • reactor: 监听和分发event
    • 处理资源池: process event, eg. read -> business -> send

  6. reactor 几种模式

    • 单 reactor 单进程/线程
    • 单 reactor 多进程/线程
    • 多 reactor 多进程/线程

  7. 具体架构(以主从reactor为例)

    • main_reactor 只负责监听连接, 将连接通过 accepter 给了 sub_reactor(可以多个)
    • sub_reactor 来负责 read + 业务逻辑 + send 流程
