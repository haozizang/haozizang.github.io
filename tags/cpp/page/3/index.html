内存对齐

  • 2022-07-25
    1. cpp
  1. sizeof 的实质
  • size of occupied memory for the instance of certain type.
    • 因此类中的函数对象不占用内存

  1. union 对齐规则

    • 不补齐, 直接按照最大的对象为尺寸

  2. struct 对齐规则

    • 计算原则
      • 原则一: 元素放置的位置一定会在自己宽度的整数倍上开始（以结构体变量首地址为0计算）。
      • 原则二、经过原则一之后，检查计算出的存储单元是否为alignof的整数倍；若不是，则补齐为它的整数倍。
      • NOTE: function 不占用内存!
    • alignof / sizeof 对比
      • sizeof 返回该结构的内存占用大小; alignof 返回对齐大小标准 min（其基本元素对齐大小, 预设对齐大小）;
    • 结构体的对齐大小:
    • 假设，无#pragma pack()情况下某结构体的对齐大小为x1; 设存在#pragma pack(z)指令，某结构体的对齐大小为x; 则 x = min(x1, z);
      • alignof(A) = min(a_default, z);
    • 某个元素a(不管是结构体还是普通元素)的sizeof()大小一定是alignof(a)的整数倍。
    • 结构体的内部元素的偏移量一定是min（其基本元素对齐大小, 预设对齐大小）的整数倍。
    • 位域
      • 若相邻位域字段类型相同:
        • 位宽和小于等于类型sizeof: 紧邻前一个字段存储
        • 位宽和大于类型sizeof: 在一个新的存储单元存

  3. 对齐相关方法

    • alignof(member); : 返回对齐大小
    • alignas(4);
    • #pragma pack(4);
    • offsetof(A, member)

  1. 快排 vs 堆排

    • 都是每次将一个数的位置固定下来; 但快排时pivot(随机), 堆排是极值

  2. 堆排序

虚拟内存

  • 2022-07-24
    1. cpp

  1. 作用

    • 让程序可以拥有超过物理内存大小的可用空间
    • 让每个程序认为自己有一片连续内存空间

  2. 原理

    • 定义了一个连续的虚拟地址空间(来连接起分散的物理内存空间)
      • MMU(内存管理单元)虚拟寻址: 虚拟地址 -> 物理地址 -> 内存
    • 把内存扩展到硬盘空间: 虚拟地址指向磁盘?
    • 不同进程使用的虚拟地址相互隔离

  3. 页面置换算法的作用?

    • 地址映射时, 若目标内容不在内存中, 则需要淘汰内存的一页, 从磁盘中载入

  4. 常见的页面置换算法有哪些?

    • 最佳页面置换算法(OPT)
    • 先进先出(FIFO)
    • 最近未使用(LRU: least currently used)
    • 最少使用页面(LFU: least frequently used)
