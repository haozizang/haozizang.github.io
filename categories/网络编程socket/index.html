socket

  • 2022-07-27
    1. 网络编程socket

  1. unix 中, 各种硬件, 接口甚至连接都被看成文件

    • 核心: 用 socket 创建了连接, 它的返回值就是 fd, 然后按照文件操作来处理网络连接

  2. windows中 socket对应的叫做 文件句柄

    • windows把socket当作连接, 不当作文件

  3. internet socket

    • 根据传输方式
      1. 流格式(SOCK_STREAM)
        • 特点
          • TCP(transmission control protocol)
          • 面向连接套接字
          • 可靠,双向,顺序;; 类比传送带
          • 发收不同步(可一个数据分开发); 接收端缓冲区可一次全读, 也可分读
        • eg.
          • http
          • FTC
      2. 数据报格式套接字(SOCK_DGRAM)
        • 特点
          • UDP(User datagram protocol)
          • 无连接套接字
          • 只传不校验, 可丢失/损毁, 不保证顺序
          • 限制每次传输大小
          • 发收同步
        • eg.
          • 视频聊天
          • DNS

  4. 无连接/面向连接

    • 无连接
      • 每个数据包在网络可以选多条不同路径(因此顺序,损丢不同!)
      • 包只包含长度/校验信息
    • 面向连接:
      • 通信前先确定一条路径(由路由记录)
      • 顺序保证: 发送后收到确认msg才发下一个
      • TCP数据量比UDP也多很多:
        • 除了传输内容数据包外, 确认数据/连接/断开包等, 导致包个数多
        • 每个包size也大:
          • 源/目标端口
          • 序号, 确认信号, 数据偏移, 控制标志(ACK URG PSH RST) etc
