进程间通信

  • 2022-07-28
    1. linux

  1. 匿名管道

    • 半双工, 单向
    • 必须亲缘关系的进程间
    • eg.
    • 步骤
      • 父进程创建管道，得到两个⽂件描述符指向管道的两端
      • 父进程fork出子进程，⼦进程也有两个⽂件描述符指向同⼀管道。
      • 父进程关闭fd[0],子进程关闭fd[1]，即⽗进程关闭管道读端,⼦进程关闭管道写端（因为管道只支持单向通信）。⽗进程可以往管道⾥写,⼦进程可以从管道⾥读,管道是⽤环形队列实现的,数据从写端流⼊从读端流出,这样就实现了进程间通信。

  2. 高级管道通信(popen)

    • 将B程序在A中启动, 算作子进程

  3. 有名管道(named pipe)

    • 半双工, 单向
    • 不必亲缘关系的进程

  4. 消息队列

    • 消息队列是由消息的链表，存放在内核中并由消息队列标识符标识
    • 消息队列克服了信号传递信息少、管道只能承载无格式字节流以及缓冲区大小受限等缺点

  5. 信号量通信(semophore)

    • 信号量是一个计数器，可以用来控制多个进程对共享资源的访问
    • 它常作为一种锁机制，防止某进程正在访问共享资源时，其他进程也访问该资源
    • 主要作为进程间以及同一进程内不同线程之间的同步手段

  6. 共享内存通信

    • 共享内存是最快的 IPC 方式
    • 共享内存就是映射一段能被其他进程所访问的内存，这段共享内存由一个进程创建，但多个进程都可以访问
    • 多与信号量配合使用来实现进程间的同步

  7. 套接字通信

    • 它可用于不同机器间的进程通信

socket

  • 2022-07-27
    1. 网络编程socket

  1. unix 中, 各种硬件, 接口甚至连接都被看成文件

    • 核心: 用 socket 创建了连接, 它的返回值就是 fd, 然后按照文件操作来处理网络连接

  2. windows中 socket对应的叫做 文件句柄

    • windows把socket当作连接, 不当作文件

  3. internet socket

    • 根据传输方式
      1. 流格式(SOCK_STREAM)
        • 特点
          • TCP(transmission control protocol)
          • 面向连接套接字
          • 可靠,双向,顺序;; 类比传送带
          • 发收不同步(可一个数据分开发); 接收端缓冲区可一次全读, 也可分读
        • eg.
          • http
          • FTC
      2. 数据报格式套接字(SOCK_DGRAM)
        • 特点
          • UDP(User datagram protocol)
          • 无连接套接字
          • 只传不校验, 可丢失/损毁, 不保证顺序
          • 限制每次传输大小
          • 发收同步
        • eg.
          • 视频聊天
          • DNS

  4. 无连接/面向连接

    • 无连接
      • 每个数据包在网络可以选多条不同路径(因此顺序,损丢不同!)
      • 包只包含长度/校验信息
    • 面向连接:
      • 通信前先确定一条路径(由路由记录)
      • 顺序保证: 发送后收到确认msg才发下一个
      • TCP数据量比UDP也多很多:
        • 除了传输内容数据包外, 确认数据/连接/断开包等, 导致包个数多
        • 每个包size也大:
          • 源/目标端口
          • 序号, 确认信号, 数据偏移, 控制标志(ACK URG PSH RST) etc

reactor

  • 2022-07-27
    1. cpp

  1. 阻塞/非阻塞

    • 无数据时:
      • 阻塞: read 时无数据则等待
      • 非阻塞(reactor): read 时无数据立即返回; 通过不断轮询来获取数据
    • 有数据时:
      • 二者都是同步

  2. 异步 I/O (proactor采用它)

    • aio_read 立即返回
    • 数据准备好(内核自动将数据从内核空间拷贝到用户空间)后通知

  3. reactor vs proactor

    • Reactor是非阻塞同步网络模式，感知的是就绪可读写事件
    • Proactor是异步网络模式，感知的是已完成的读写事件

  4. reactor 模式的应用

    • redis
    • nginx
    • netty

  5. reactor组成

    • reactor: 监听和分发event
    • 处理资源池: process event, eg. read -> business -> send

  6. reactor 几种模式

    • 单 reactor 单进程/线程
    • 单 reactor 多进程/线程
    • 多 reactor 多进程/线程

  7. 具体架构(以主从reactor为例)

    • main_reactor 只负责监听连接, 将连接通过 accepter 给了 sub_reactor(可以多个)
    • sub_reactor 来负责 read + 业务逻辑 + send 流程
