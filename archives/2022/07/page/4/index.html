内存对齐

  • 2022-07-25
    1. cpp
  1. sizeof 的实质
  • size of occupied memory for the instance of certain type.
    • 因此类中的函数对象不占用内存

  1. union 对齐规则

    • 不补齐, 直接按照最大的对象为尺寸

  2. struct 对齐规则

    • 计算原则
      • 原则一: 元素放置的位置一定会在自己宽度的整数倍上开始（以结构体变量首地址为0计算）。
      • 原则二、经过原则一之后，检查计算出的存储单元是否为alignof的整数倍；若不是，则补齐为它的整数倍。
      • NOTE: function 不占用内存!
    • alignof / sizeof 对比
      • sizeof 返回该结构的内存占用大小; alignof 返回对齐大小标准 min（其基本元素对齐大小, 预设对齐大小）;
    • 结构体的对齐大小:
    • 假设，无#pragma pack()情况下某结构体的对齐大小为x1; 设存在#pragma pack(z)指令，某结构体的对齐大小为x; 则 x = min(x1, z);
      • alignof(A) = min(a_default, z);
    • 某个元素a(不管是结构体还是普通元素)的sizeof()大小一定是alignof(a)的整数倍。
    • 结构体的内部元素的偏移量一定是min（其基本元素对齐大小, 预设对齐大小）的整数倍。
    • 位域
      • 若相邻位域字段类型相同:
        • 位宽和小于等于类型sizeof: 紧邻前一个字段存储
        • 位宽和大于类型sizeof: 在一个新的存储单元存

  3. 对齐相关方法

    • alignof(member); : 返回对齐大小
    • alignas(4);
    • #pragma pack(4);
    • offsetof(A, member)

pid

  • 2022-07-24
    1. control

  1. u(t) = Kp * e(t) + Ki * ∫e(t)dt + Kd * (de(t) / dt)

  2. Kp

    • proportional gain
    • 作用
      • 达到目标值更快
      • 减小 稳态误差(steady-state error)
      • 增大 overshoot

  3. Kd

    • derivative term
    • 作用
      • 根据error的变化率快慢来调整相应的输入量, 减小 overshoot

  4. Ki

    • derivative integral
    • 作用
      • 减小 稳态误差(steady-state error)
      • 不利点: 让系统变得迟缓, 多震荡(因为当error变号, )

  5. 响应的几个指标

    • fast rise time
    • minimal overshoot
    • zero steady-state error
