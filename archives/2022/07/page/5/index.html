虚拟内存

  • 2022-07-24
    1. cpp

  1. 作用

    • 让程序可以拥有超过物理内存大小的可用空间
    • 让每个程序认为自己有一片连续内存空间

  2. 原理

    • 定义了一个连续的虚拟地址空间(来连接起分散的物理内存空间)
      • MMU(内存管理单元)虚拟寻址: 虚拟地址 -> 物理地址 -> 内存
    • 把内存扩展到硬盘空间: 虚拟地址指向磁盘?
    • 不同进程使用的虚拟地址相互隔离

  3. 页面置换算法的作用?

    • 地址映射时, 若目标内容不在内存中, 则需要淘汰内存的一页, 从磁盘中载入

  4. 常见的页面置换算法有哪些?

    • 最佳页面置换算法(OPT)
    • 先进先出(FIFO)
    • 最近未使用(LRU: least currently used)
    • 最少使用页面(LFU: least frequently used)

源码-容器

  1. stack

    • 底层默认 Sequence 是从 deque 修改过来的
    • 不提供迭代器

  2. queue

    • 不提供迭代器
    • 底层默认 Sequence 是从 deque 修改过来的

  3. map/set

    • 递增 -> 红黑树实现
    • set 的迭代器是 constance_iterators, 不能修改键
    • set 保存了两份(key/value 相等? why? )

  4. 树节点

    1. 无序树/有序树
      • 任意节点的子节点 有/无 顺序关系
    2. 二叉树
      • 树的任意节点至多包含两棵子树
      • 遍历方法: 前序/中序/后序/层次
      • 完美二叉树: 叶子节点都在同一层; 且除叶子节点外所有节点都有俩子节点
      • 完全二叉树(<=完美二叉树): 除最深层(d层)外的所有节点构成满二叉树, 且d层所有节点从左向右连续紧密排列
      • 完满二叉树(<=完美二叉树): 且除叶子节点外所有节点都有俩子节点
    3. 霍夫曼树
      • 加权路径最短的二叉树
    4. 二叉查找树(BST)
      • 左nodes < 根 node < 右nodes
      • 递归性: 左右两个子树也都是BST
      • 无相等节点
    5. 平衡二叉树
      • 左右两个子树高度差不超过 1
      • 递归性: 左右两个子树都是平衡二叉树
    6. AVL树
      • BST
      • 平衡: 左右子树高度差不超过 1
      • 使用场景
        • 增删少, 查找多
        • 旋转: 降低树的高度, 再平衡
    7. 红黑树
      • BST
      • 节点非黑即红
      • 根节点为黑
      • 红色节点两个子节点都是黑色
      • 从任一节点到其每个叶子的所有路径都包含相同数目的黑色节点
      • 这些约束确保：从根到叶子的最长的可能路径不多于最短的可能路径的两倍长
      • 使用场景
        • 增删也多, 查找多
        • STL中的 map / set; linux 进程调度(completely fair scheduler)
        • epoll 内核中用 RBT 管理事件块
        • nginx 红黑树管理 timer
      • 和 AVL对比:
        • 不严格平衡, 增删快;
        • 旋转次数3此内解决;
        • 查找 AVL 快
    8. 伸展树
      • BST
      • O(logn) 内完成增删查
    9. B-tree (m阶)
      • 节点最多m个子节点
      • 根节点(若非叶)最少2个子
      • 非叶子节点(非根)最少 [m/2向上取整] 个子节点
      • 非根子节点度数k为: m/2 <= k <= m
      • 非根子节点 键数 j: j = k - 1
      • 叶子节点位于同一层(平衡因子 = 0)
      • 特性
        • 自平衡
        • 多路(降低了深度): 适合磁盘数据读取(IO次数少)
        • O(logN)
    10. B+
      • 数据都放在叶子节点, 非叶子节点只存索引
    11. Trie树(字典树)
      • 根节点不包含字符，除根节点外每一个节点都只包含一个字符
      • 从根节点到某一节点，路径上经过的字符连接起来，为该节点对应的字符串
      • 每个节点的所有子节点包含的字符都不相同
      • 应用
        • 字符串查找
        • 搜索引擎

    1. cpp

  1. 数组作为函数对象时, 退化为指针

  2. 返回值不能区分重载函数，只有形参的个数、类型或者顺序可以(const 修饰的 this 也可以)

  3. void func() const {..} 这个const 修饰的是隐式第一形参 this 指针!

  4. 常量折叠

    • const 常量在编译阶段被替换, 因此在程序中(强行)修改它会不起作用

  5. struct 对齐规则

    • 计算原则

      • 原则一: 元素放置的位置一定会在自己宽度的整数倍上开始（以结构体变量首地址为0计算）。

      • 原则二、经过原则一之后，检查计算出的存储单元是否为alignof的整数倍；若不是，则补齐为它的整数倍。

    • alignof / sizeof 对比
      • sizeof 返回该结构的内存占用大小; alignof 返回对齐大小标准 min（其基本元素对齐大小, 预设对齐大小）;
    • 结构体的对齐大小:
    • 假设，无#pragma pack()情况下某结构体的对齐大小为x1; 设存在#pragma pack(z)指令，某结构体的对齐大小为x; 则 x = min(x1, z);
      • alignof(A) = min(a_default, z);
    • 某个元素a(不管是结构体还是普通元素)的sizeof()大小一定是alignof(a)的整数倍。
    • 结构体的内部元素的偏移量一定是min（其基本元素对齐大小, 预设对齐大小）的整数倍。
    • 位域
      • 若相邻位域字段类型相同:
        • 位宽和小于等于类型sizeof: 紧邻前一个字段存储
        • 位宽和大于类型sizeof: 在一个新的存储单元存

  6. 对齐相关方法

    • alignof(member); : 返回对齐大小
    • alignas(4);
    • #pragma pack(4);
    • offsetof(A, member)

净资产收益率

  • 2022-07-21
    1. inv

ROE(return on equity)

  1. 净资产收益率 分两种

    • 加权平均净资产收益率=报告期净利润/年平均净资产
    • 全面摊薄净资产收益率=报告期净利润/期末净资产

  2. 长期持股的收益率真的接近ROE吗?

    • 长期持股，不分红或少分红情况下，其持股收益接近于净资产收益率
    • 高ROE的公司如果分红(90%)，对收益率影响非常大
    • 低PB + 少分红 -> 此结论接近成立
    • PB小于1时，分红率提高会提升回报率，ROE越高影响越大。

  1. 覆盖

    1. 函数签名

      • 函数名
      • 参数(类型/个数/顺序)
      • 不包括返回值

    2. 重载 : 复用同一个函数名

      • 同一个类中(水平关系)
      • 要求参数列表不同
      • 重载不能用返回值来区别

    3. 覆盖

      • 子类和父类之间的关系(垂直关系)
      • 要求函数签名一致
      • 要求基类中被覆盖的函数加 virtual(虚函数的实现)
      • 基类析构函数定义为virtual后也可覆盖
      • 子类中方法的权限要 >= 父类中被覆盖的方法

  2. 逗号运算符，返回最右边的值。

    • (2,3)的值是3
    • ((2,3),4)的值是4

  3. 对于内置类型而言，new仅仅是分配内存，除非后面显示加(),相当于调用它的构造函数，对于自定义类型而言，只要一调用new，那么编译器不仅仅给它分配内存，还调用它的默认构造函数初始化，即使后面没有加()
