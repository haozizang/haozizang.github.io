源码-容器

  1. stack

    • 底层默认 Sequence 是从 deque 修改过来的
    • 不提供迭代器

  2. queue

    • 不提供迭代器
    • 底层默认 Sequence 是从 deque 修改过来的

  3. map/set

    • 递增 -> 红黑树实现
    • set 的迭代器是 constance_iterators, 不能修改键
    • set 保存了两份(key/value 相等? why? )

  4. 树节点

    1. 无序树/有序树
      • 任意节点的子节点 有/无 顺序关系
    2. 二叉树
      • 树的任意节点至多包含两棵子树
      • 遍历方法: 前序/中序/后序/层次
      • 完美二叉树: 叶子节点都在同一层; 且除叶子节点外所有节点都有俩子节点
      • 完全二叉树(<=完美二叉树): 除最深层(d层)外的所有节点构成满二叉树, 且d层所有节点从左向右连续紧密排列
      • 完满二叉树(<=完美二叉树): 且除叶子节点外所有节点都有俩子节点
    3. 霍夫曼树
      • 加权路径最短的二叉树
    4. 二叉查找树(BST)
      • 左nodes < 根 node < 右nodes
      • 递归性: 左右两个子树也都是BST
      • 无相等节点
    5. 平衡二叉树
      • 左右两个子树高度差不超过 1
      • 递归性: 左右两个子树都是平衡二叉树
    6. AVL树
      • BST
      • 平衡: 左右子树高度差不超过 1
      • 使用场景
        • 增删少, 查找多
        • 旋转: 降低树的高度, 再平衡
    7. 红黑树
      • BST
      • 节点非黑即红
      • 根节点为黑
      • 红色节点两个子节点都是黑色
      • 从任一节点到其每个叶子的所有路径都包含相同数目的黑色节点
      • 这些约束确保：从根到叶子的最长的可能路径不多于最短的可能路径的两倍长
      • 使用场景
        • 增删也多, 查找多
        • STL中的 map / set; linux 进程调度(completely fair scheduler)
        • epoll 内核中用 RBT 管理事件块
        • nginx 红黑树管理 timer
      • 和 AVL对比:
        • 不严格平衡, 增删快;
        • 旋转次数3此内解决;
        • 查找 AVL 快
    8. 伸展树
      • BST
      • O(logn) 内完成增删查
    9. B-tree (m阶)
      • 节点最多m个子节点
      • 根节点(若非叶)最少2个子
      • 非叶子节点(非根)最少 [m/2向上取整] 个子节点
      • 非根子节点度数k为: m/2 <= k <= m
      • 非根子节点 键数 j: j = k - 1
      • 叶子节点位于同一层(平衡因子 = 0)
      • 特性
        • 自平衡
        • 多路(降低了深度): 适合磁盘数据读取(IO次数少)
        • O(logN)
    10. B+
      • 数据都放在叶子节点, 非叶子节点只存索引
    11. Trie树(字典树)
      • 根节点不包含字符，除根节点外每一个节点都只包含一个字符
      • 从根节点到某一节点，路径上经过的字符连接起来，为该节点对应的字符串
      • 每个节点的所有子节点包含的字符都不相同
      • 应用
        • 字符串查找
        • 搜索引擎

  1. 覆盖

    1. 函数签名

      • 函数名
      • 参数(类型/个数/顺序)
      • 不包括返回值

    2. 重载 : 复用同一个函数名

      • 同一个类中(水平关系)
      • 要求参数列表不同
      • 重载不能用返回值来区别

    3. 覆盖

      • 子类和父类之间的关系(垂直关系)
      • 要求函数签名一致
      • 要求基类中被覆盖的函数加 virtual(虚函数的实现)
      • 基类析构函数定义为virtual后也可覆盖
      • 子类中方法的权限要 >= 父类中被覆盖的方法

  2. 逗号运算符，返回最右边的值。

    • (2,3)的值是3
    • ((2,3),4)的值是4

  3. 对于内置类型而言，new仅仅是分配内存，除非后面显示加(),相当于调用它的构造函数，对于自定义类型而言，只要一调用new，那么编译器不仅仅给它分配内存，还调用它的默认构造函数初始化，即使后面没有加()

    • 但编译器自动优化, 不绝对

  6. 符合语句: {…}

  7. printf(“%-4.3d”, 12345.78901);

    • -: 左对齐
    • 4: 全长度, 不够补空

  8. 访问权限

    • class: pri

  9. 纯虚函数

  3. void func() const {..} 这个const 修饰的是隐式第一形参 this 指针!

  4. 常量折叠

    • const 常量在编译阶段被替换, 因此在程序中(强行)修改它会不起作用

  5. struct 对齐规则

    • 计算原则

      • 原则一: 元素放置的位置一定会在自己宽度的整数倍上开始（以结构体变量首地址为0计算）。

      • 原则二、经过原则一之后，检查计算出的存储单元是否为alignof的整数倍；若不是，则补齐为它的整数倍。

    • alignof / sizeof 对比
      • sizeof 返回该结构的内存占用大小; alignof 返回对齐大小标准 min（其基本元素对齐大小, 预设对齐大小）;
    • 结构体的对齐大小:
    • 假设，无#pragma pack()情况下某结构体的对齐大小为x1; 设存在#pragma pack(z)指令，某结构体的对齐大小为x; 则 x = min(x1, z);
      • alignof(A) = min(a_default, z);
    • 某个元素a(不管是结构体还是普通元素)的sizeof()大小一定是alignof(a)的整数倍。
    • 结构体的内部元素的偏移量一定是min（其基本元素对齐大小, 预设对齐大小）的整数倍。
    • 位域
      • 若相邻位域字段类型相同:
        • 位宽和小于等于类型sizeof: 紧邻前一个字段存储
        • 位宽和大于类型sizeof: 在一个新的存储单元存

  6. 对齐相关方法

    • alignof(member); : 返回对齐大小
    • alignas(4);
    • #pragma pack(4);
    • offsetof(A, member)

socket

  • 2022-07-29
    1. cpp

  1. unix 中, 各种硬件, 接口甚至连接都被看成文件

    • 核心: 用 socket 创建了连接, 它的返回值就是 fd, 然后按照文件操作来处理网络连接

  2. windows中 socket对应的叫做 文件句柄

    • windows把socket当作连接, 不当作文件

  3. internet socket

    • 根据传输方式
      1. 流格式(SOCK_STREAM)
        • 特点
          • TCP(transmission control protocol)
          • 面向连接套接字
          • 可靠,双向,顺序;; 类比传送带
          • 发收不同步(可一个数据分开发); 接收端缓冲区可一次全读, 也可分读
        • eg.
          • http
          • FTC
      2. 数据报格式套接字(SOCK_DGRAM)
        • 特点
          • UDP(User datagram protocol)
          • 无连接套接字
          • 只传不校验, 可丢失/损毁, 不保证顺序
          • 限制每次传输大小
          • 发收同步
        • eg.
          • 视频聊天
          • DNS

  4. 无连接/面向连接

    • 无连接
      • 每个数据包在网络可以选多条不同路径(因此顺序,损丢不同!)
      • 包只包含长度/校验信息
    • 面向连接:
      • 通信前先确定一条路径(由路由记录)
      • 顺序保证: 发送后收到确认msg才发下一个
      • TCP数据量比UDP也多很多:
        • 除了传输内容数据包外, 确认数据/连接/断开包等, 导致包个数多
        • 每个包size也大:
          • 源/目标端口
          • 序号, 确认信号, 数据偏移, 控制标志(ACK URG PSH RST) etc

进程间通信

  • 2022-07-29
    1. cpp

  1. 匿名管道

    • 半双工, 单向
    • 必须亲缘关系的进程间
    • eg.
    • 步骤
      • 父进程创建管道，得到两个⽂件描述符指向管道的两端
      • 父进程fork出子进程，⼦进程也有两个⽂件描述符指向同⼀管道。
      • 父进程关闭fd[0],子进程关闭fd[1]，即⽗进程关闭管道读端,⼦进程关闭管道写端（因为管道只支持单向通信）。⽗进程可以往管道⾥写,⼦进程可以从管道⾥读,管道是⽤环形队列实现的,数据从写端流⼊从读端流出,这样就实现了进程间通信。

  2. 高级管道通信(popen)

    • 将B程序在A中启动, 算作子进程

  3. 有名管道(named pipe)

    • 半双工, 单向
    • 不必亲缘关系的进程

  4. 消息队列

    • 消息队列是由消息的链表，存放在内核中并由消息队列标识符标识
    • 消息队列克服了信号传递信息少、管道只能承载无格式字节流以及缓冲区大小受限等缺点

  5. 信号量通信(semophore)

    • 信号量是一个计数器，可以用来控制多个进程对共享资源的访问
    • 它常作为一种锁机制，防止某进程正在访问共享资源时，其他进程也访问该资源
    • 主要作为进程间以及同一进程内不同线程之间的同步手段

  6. 共享内存通信

    • 共享内存是最快的 IPC 方式
    • 共享内存就是映射一段能被其他进程所访问的内存，这段共享内存由一个进程创建，但多个进程都可以访问
    • 多与信号量配合使用来实现进程间的同步

  7. 套接字通信

    • 它可用于不同机器间的进程通信

内存池

  • 2022-07-29
    1. cpp

  1. 内存池

    • 优点
      • 减少new/delete次数
      • 避免内存碎片
    • 方法
      • 一次性获取大块的资源, 然后进程自己管理内存的释放和申请
      • 提前申请
      • 重复利用
      • 减少了 new/delete 的次数

  2. 内存申请

    • C(malloc/free) C++(new/delete) 会申请内存块, 切割, 剩余放回内存链表中
    • 内存碎片: 切割后不在连续的内存
    • 问题: 内存总量够但却无法找到大的连续内存块
    • 解决方法: 内存池

  3. STL 容器的 new/delete 其实底层也有 内存池优化

  1. 市现率 = 股价 / 每股现金流量

    • 分析和评估股票的价格水平和风险水平?
    • 市现率越小, 表明公司每股现金增加额越多, 经营越健康

    • 市销率越低，说明该公司股票目前的投资价值越大。
    • 意义: 获得单位的销售收入需要投入的金额
